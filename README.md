# Data Quality Management - Expert AI Implementation

This application demonstrates **best practices** for a Data Quality Management system:

## Key Differences from LOCAL AI Version

### Architecture
- **Proper lifespan management** with async context managers
- **Dependency injection** for database sessions
- **Global exception handling** with logging
- **Pydantic Settings** for configuration validation
- **Type hints** throughout the codebase

### Security
- **SQL injection prevention** with parameterized queries
- **Table whitelist** to prevent unauthorized access
- **Input validation** with Pydantic models
- **Proper CORS configuration**

### Code Quality
- **Dataclasses** for structured data
- **Enums** for rule types and severities
- **Comprehensive error handling**
- **Logging** for debugging and monitoring

### Features
- Full data profiling (nulls, uniqueness, distribution, statistics)
- Rule suggestion based on profiling results
- Multiple rule types (null, unique, range, pattern)
- AI-powered root cause analysis using LOCAL AI (RTX 5090)
- React frontend with dark theme

## Setup

```bash
# Backend
cd backend
pip install -r requirements.txt
uvicorn app.main:app --port 8000

# Frontend
cd frontend
npm install
npm run dev  # Runs on port 3001
```

## Architecture

- **Backend**: FastAPI + SQLAlchemy async + asyncpg
- **Frontend**: React 18 + Vite
- **Database**: PostgreSQL (Northwind)
- **AI**: LOCAL AI models (RTX 5090 Qwen2.5-32B)
